PostgreSQL View nima?

O‘zbekcha tushuntirish:
PostgreSQL’da View — bu saqlangan SQL so‘rovi bo‘lib, u bazadagi jadvallardan yoki boshqa ko‘rinishlardan olingan ma’lumotlarni ifodalaydi. View’lar virtual jadval sifatida ishlaydi va ularning o‘zida hech qanday ma’lumot saqlanmaydi, balki asosiy jadvallardan so‘rov bajarilganda natijalarni qaytaradi.

View’ning afzalliklari:

  • Murakkab so‘rovlarni soddalashtirish – Ko‘p qatlamli yoki murakkab so‘rovlarni bitta View orqali ishlatish mumkin.
  • Ma’lumot xavfsizligi – Foydalanuvchilarga faqat muayyan ustun yoki qatorlarga ruxsat berish uchun ishlatiladi.
  • Ma’lumot mustahkamligini ta’minlash – Ma’lumotlar bazasi sxemasini o‘zgartirmasdan turib, eski so‘rovlarni saqlab qolish imkonini beradi.
  • Oqimni tezlashtirish – Ba’zan View’lar optimallashtirilgan shaklda saqlanib, tezkor natijalarni qaytarishi mumkin.

Misol:
Faraz qilaylik, bizda employees jadvali bor. Quyidagi View xodimlarning faqat ID, ismi va ish haqi haqidagi ma’lumotlarni chiqaradi:

CREATE VIEW employee_salaries ASSELECT id, name, salaryFROM employees;

Endi, bu View’ni xuddi oddiy jadval kabi ishlatish mumkin:

SELECT * FROM employee_salaries;

What is a PostgreSQL View?

In PostgreSQL, a View is a stored SQL query that represents data from one or more tables or other views. Views act as virtual tables, meaning they don’t store data themselves but return results dynamically from underlying tables when queried.

Advantages of Views:

  • Simplifies complex queries – You can use Views to manage multi-layered or complex queries easily.
  • Enhances data security – Views can restrict access to specific columns or rows for certain users.
  • Ensures data consistency – It allows keeping old queries functional even if the database schema changes.
  • Optimized performance – In some cases, Views can be materialized to improve query speed.

Example:
Suppose we have an employees table. The following View displays only the id, name, and salary columns:

CREATE VIEW employee_salaries ASSELECT id, name, salaryFROM employees;

Now, we can use the View like a regular table:

SELECT * FROM employee_salaries;
